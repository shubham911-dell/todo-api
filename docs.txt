# ÔøΩ Todo App Data Flow

This document explains how data moves between the Frontend (Browser) and Backend (Server).

## 1. üöÄ Page Load (Initialization)
1. **Browser** loads `index.html`.
2. **Script** (`script.js`) runs automatically and calls `loadTodos()`.
3. **API Request**: Sends a `GET` request to `http://localhost:5000/api/todos`.
4. **Server**: Queries the database for all todos and returns them as JSON.
5. **UI Update**: The script receives the data and renders the list (`<ul>`) on the screen.

---

## 2. ‚ûï Add Todo
1. **User Action**: Types a todo and clicks the **Add** button.
2. **Script**: 
   - Reads the input value.
   - Sends a `POST` request to `http://localhost:5000/api/todos`.
   - **Body**: `{"title": "New Todo Item"}`
3. **Server**: SQL `INSERT` command adds valid data to the database.
4. **UI Update**: The script calls `loadTodos()` again to refresh the list with the new item.

---

## 3. ‚ùå Delete Todo
1. **User Action**: Clicks the **Delete** button next to a specific item.
2. **Script**: 
   - Captures the ID of that todo (e.g., `5`).
   - Sends a `DELETE` request to `http://localhost:5000/api/todos/5`.
3. **Server**: SQL `DELETE` command removes the row with `id = 5`.
4. **UI Update**: The script calls `loadTodos()` to refresh the list, removing the deleted item.
